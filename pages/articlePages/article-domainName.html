Domain Name

Aspectos Importantes:

1 - Ninguém é dono de um nome de domínio:

Você não pode "comprar um nome de domínio". Em vez disso, você paga pelo direito de usar um nome de domínio por um ou mais anos. Você pode renovar seu direito, e sua renovação tem prioridade sobre os pedidos de outras pessoas. Mas você nunca é o proprietário do nome de domínio.

Processo de Registro de um domínio:

  • Escolher o nome e o TLD;
  • Verificar se está disponível;
  • Pagar pelo período desejado;
  • Fornecer informações do titular;
  • Configurar seu domínio;
  • Renovar para Manter

2 - Whois:

Empresas chamadas registradores usam registros de nomes de domínio para rastrear informações técnicas e administrativas que conectam você ao seu nome de domínio. Acesse o site de um registrador de nomes de domínio. A maioria deles oferece um serviço "whois" que informa se um nome de domínio está disponível.

3 - DNSchecker:

Quando você registra um novo domínio e configura seu DNS (por exemplo, apontando para um servidor de hospedagem), essa informação precisa se espalhar pela rede global de servidores DNS. Esse processo de atualização e sincronização é chamado de propagação DNS. Ferramentas como o DNSchecker monitoram a propagação do domínio pelos servidores globais.

DNSchecker -> https://dnschecker.org/

Labels (Rótulos):

Um rótulo é uma sequência de caracteres com comprimento entre um e sessenta e três caracteres, contendo apenas as letras `A`até `Z`, os dígitos `0`até `9`e o caractere '-' (que pode não ser o primeiro ou o último caractere do rótulo). São as partes de um nome de domínio separadas por pontos (`.`). Os labels são usados para organizar hierarquicamente os nomes dos sites, da parte mais geral (à direita) até a mais específica (à esquerda). São as partes fundamentais que constituem um domínio.

EX:

  • google.comcode> -> 2 labels
  • mail.google.comcode> -> 3 labels
  • www.uol.com.brcode> -> 4 labels
  • login.secure.mybank.co.ukcode> -> 5 labels
  • a.b.c.d.e.f.g.comcode> -> 8 labels
  • portal.ifsp.edu.brcode> -> 4 labels

3LD / SubDomains:

Subdomínios são divisões ou extensões de um domínio principal na estrutura de endereços da internet (DNS – Domain Name System). Eles funcionam como uma forma de organizar e separar seções diferentes de um site ou serviço dentro do mesmo domínio principal.

EX:

  • maps.google.comcode> -> Sub Domain = maps
  • translate.google.comcode> -> Sub Domain = translate
  • music.amazon.comcode> -> Sub Domain = music
  • sellercentral.amazon.comcode> -> Sub Domain = sellercentral
  • esporte.uol.com.brcode> -> Sub Domain = esporte
  • economia.uol.com.brcode> -> Sub Domain = economia
  • support.microsoft.comcode> -> Sub Domain = support
  • gshow.globo.comcode> -> Sub Domain = gshow

2LD / SLDs (Second Level Domains):

É a parte do domínio que vem logo antes do TLD. Identifica a entidade principal do domínio, como uma empresa, organização, marca ou instituição. É o nome que você registra quando compra um domínio (ex: você registra `meusite.com` → o SLD é `meusite`).

EX:

  • goolge.comcode> -> SLD = google
  • mail.google.comcode> -> SLD = google
  • www.uol.com.brcode> -> SLD = uol
  • support.mozilla.orgcode> -> SLD = mozilla
  • blog.nasa.govcode> -> SLD = nasa
  • api.github.devcode> -> SLD = github
  • student.portal.university.educode> -> SLD = university
  • informatics.ed.ac.ukcode> -> SLD = informatics

TLDs (Top level Domains):

Os TLDs ajudam a classificar e organizar os domínios na internet. Eles indicam o tipo de entidade (comercial, governamental, educacional, etc.) ou o país ao qual o site pertence. São divididos em: gTLDs, ccTLDs e nTLDs. Os TLDs podem conter caracteres especiais e latinos.

Lista completa de TLDs -> https://data.iana.org/TLD/tlds-alpha-by-domain.txt

gTLDs (Generic Top Level Domains):

Domínios genéricos, usados globalmente. Não exigem que os serviços web atendam a nenhum critério específico, mas alguns TLDs aplicam políticas mais rígidas para que fique mais claro qual é o seu propósito. Por exemplo:

  • .gov: Pode ser usado apenas por departamentos governamentais.
  • .edu: É para uso somente por instituições educacionais e acadêmicas.
    •

    EX:

    • `.com` -> Commercial -> Negócios, empresas, uso geral
    • `.org` -> Organization -> ONGs, projetos comunitários
    • `.net` -> Network -> Provedores de internet, uso técnico
    • `.edu` -> Education -> Instituições educacionais (restrito, principalmente nos EUA)
    • `.gov` -> Government -> Órgãos governamentais (principalmente nos EUA)
    • `.mil` -> Military -> Uso militar (exclusivo dos EUA)
    • `.int` -> International -> Organizações internacionais

    ccTLDs (Country Codes Top Level Domain):

    Domínios com código de país. Sempre com 2 letras. Representam países ou territórios.

    EX:

    • .br -> Brasil -> Sites brasileiros
    • .uk -> Reino Unido -> Sites britânicos
    • .de -> Alemanha -> Sites alemães
    • .jp -> Japão -> Sites japoneses
    • .fr -> França -> Sites franceses
    • .cn -> China -> Sites chineses
    • .ru -> Rússia -> Sites russos
    • .us -> Estados Unidos -> Uso local dentro dos EUA
    • .pt -> Portugal -> Sites portugueses
    • .au -> Austrália -> Sites australianos

    nTLDs (New Top Level Domains):

    Domínios criados após 2013 para expandir as opções. Podem ser temáticos, personalizados ou voltados a nichos. Mais flexíveis, modernos, temáticos e criativos.

    EX:

    • `.tech` -> Tecnologia -> Startups, empresas de TI
    • `.store` -> Loja online -> E-commerce
    • `.app` -> Aplicativos -> Softwares e apps móveis
    • `xyz` -> Genérico -> Alternativa criativa e flexível
    • `.online` -> Presença digital -> Sites pessoais e comerciais
    • `.blog` -> Conteúdo -> Blogs e publicações
    • `.design` -> Design -> Portfólios, agências criativas
    • `.dev` -> Desenvolvimento -> Projetos e ferramentas de desenvolvedores
    • `.io` -> (território Britânico) mas virou moda no setor tech -> Usado por startups e serviços digitais
